## Summary
Career Decision Intelligence is a secure, guided, explainable web application that helps a student move from career uncertainty to a structured decision. The MVP should be built AI-optional and deterministic-first, using profile inputs, weighted preferences, constraints, and curated career-path metadata to generate explainable recommendations, comparisons, and action plans.

## Priority build order
1. App scaffold, auth, layout shell, and design system foundation
2. Onboarding flow with name capture, profile questions, weights, and constraints
3. Seed data model for career paths and trait mappings
4. Deterministic recommendation engine and recommendation result UI
5. Shortlist, compare screen, and weight-adjustment workflow
6. Decision board and action-plan generation
7. Privacy/account controls and admin content tooling

## Key decisions made during spec

| Decision | Rationale |
|----------|-----------|
| Web app first | Fastest route to secure prototype and iteration |
| AI optional, not core | Protects trust, clarity, and builder focus |
| Deterministic scoring engine | Makes recommendations explainable and testable |
| Preferred-name capture in onboarding | Supports personal tone without overcomplicating identity |
| Curated seed data first | Avoids premature external integrations |
| Supabase/Postgres backend | Good speed/security balance for prototype |

## Risks identified
- Weak seed content can make recommendations feel shallow; mitigate with a smaller, well-authored path library.
- Product may drift into quiz territory; mitigate through strong comparison, explanation, and decision outputs.
- AI temptation may distort scope; mitigate by keeping AI out of the MVP critical path.
- Privacy trust can erode quickly; mitigate with explicit controls and minimal data collection.
